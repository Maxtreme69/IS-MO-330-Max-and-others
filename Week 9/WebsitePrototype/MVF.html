Manage Project
When users open the application for the first time, they will have to create a project. There will be a graphical interface and an input field where users can input the name of their project. Add team members to a new or existing project as well as add, delete and update projects which will be stored the web applications database.
Validation Testing
  • Test 1 - When the user inputs Team Project name and clicks create team, The project should store in the database and be displayed on the graphical user interface.
  • Test 2 - Once project is created add team member input fields should appear on the graphical user interface
  • Test 3 - When user types in team member name and clicks add, team member should be stored to that project in the database and member should appear under team project name on screen
  • Test 4 - When delete icon is clicked on project the project should be deleted from the database and disappear from graphical user interface
  • Test 5 - When delete icon is clicked on team member the project should be deleted from the database and disappear from graphical user interface

Manage Sticky Notes
The Scrum Master application will have the functionality to manage notes. Users will be able to create notes, delete notes as well as add, delete and update tasks on each sticky note.
Validation Testing
  • Test 1 - When sticky note "+" icon is clicked a new sticky note is created on scrumboard and stored in database
  • Test 2 - When "x" icon is clicked sticky note is deleted from scrumboard and is removed from database
  • Test 3 - When task information is input and add task icon is clicked task and information are saved to the database and stored on sticky note on the scrumboard.
  • Test 4 - When task delete icon is clicked, task and information is removed from sticky note and is removed from the database.
  • Test 5 - When update button is pressed task information becomes modifiable, when update button is pressed again modified task is displayed on scrumboard and saved in database.

Drag & Drop
Having the abillity for notes to be placed in different categories on the scrum board is an essential feature for the Scrum Master application. When users select notes they will have the ability to place into any of the columns of the scrum board.
Validation Testing
  • Test 1 - When cursor hovers over sticky note, sticky note highlights
  • Test 2 - When user drags note to another column the background highlights to indicate where the note can be placed on scrumboard and drops in highlighted field when user unclicks mouse
  • Test 3 - When note is moved and background is not highlighted, when mouse button is unclicked, note returns to location it was previously on the scrumboard.

Calendar
The Scrum Master application will include an interactive calendar in which users can navigate to different dates.
Validation Testing
  • Test 1 - When cursor hovers over date, date highlights.
  • Test 2 - When arrow buttons are pressed date on calendar moves from left to right showing the previous day if left button is clicked and showing the next day if the right button is clicked.
  • Test 3 - When date is clicked calendar day colour is changed to indicate date which is selected.

Progress Tracking
Scrum Master will have the ability to track the state of the scrumboard throughout different dates within the application. When users click on a date, the state of the scrumboard will display the activities how they were on that particular day.
Validation Testing
  • Test 1 - When save button icon is clicked scrumboard is saved to the database
  • Test 2 - When arrow buttons are pressed date on calendar moves from left to right showing the previous day if left button is clicked and showing the next day if the right button is clicked.
  • Test 3 - When date is clicked calendar day colour is changed to indicate date which is selected.
